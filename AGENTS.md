## Project Rules

- 使用 `bun i` 安装依赖。
- 使用尽可能现代的 JavaScript 语法，以及现代浏览器或运行环境已支持的平台能力。
- 默认使用 `const`。如果是用 `const` 定义的真正常量，名称应使用大写风格。
- 变量命名尽量精简，并使用 `snake_case`。
- 函数命名使用小写驼峰风格。
- 函数统一使用箭头函数形式定义，例如 `const funcName = ()=>{}`。
- 不要使用 `function` 声明函数。
- 连续的多个 `const` 声明在合适时应合并，例如 `const a = 1, b = 2, c = 3;`。
- `import` 时优先导入具名函数，尽量避免直接导入整个模块。
- 使用 `await`，不要使用 `.then()`。
- 优先编写纯函数，不要使用类。
- 注重代码复用，多抽取函数，避免出现大量重复或近似重复的代码结构。
- 使用现代浏览器支持的原生 CSS nesting 来减少样式冗余。
- 对于 `export default` 导出的函数或变量，除非存在 `import.meta.main` 的调用需求，否则不要先声明 `const` 再默认导出。
- 修改完成后运行 `./build.sh`，并修复出现的错误。
